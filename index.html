Model Portfolio — Dorman / Arca Framework (Educational Only)
"Own tokens that make money & buy back their own tokens with the profits" — Jeff Dorman
▶
Arca's CIO advocates a fundamental, bottoms-up approach: prioritize revenue-generating protocols with real yield, active buybacks, and sustainable moats. Alt season is dead — only projects with tangible business models survive. The allocation below reflects Dorman's 2025 framework: overweight Deep Value (low P/E + buybacks), core allocation to Fair Value (proven protocols), reduced exposure to Growth (pay only for dominance), and minimal or zero for Overheated.
Dorman's Core Rules
1. Revenue > Narrative — only invest in protocols generating real fees
2. Buyback = conviction — teams buying their own token signal belief
3. Circulating MC, not FDV — valuation based on actual float
4. Diversify by function — DEX, lending, RWA, infrastructure
5. Risk-adjust by P/E zone — heavier in Deep Value, lighter in Premium
Allocation Logic
Deep Value (P/E <10×) → 25-35% each — max conviction
Fair Value (10-18×) → 10-20% each — core holdings
Growth (18-30×) → 5-10% each — selective exposure
Premium (30-60×) → 0-5% — only with exceptional moat
Overheated (>60×) → 0% — take profits, redeploy
Allocation weights computed from live P/E zone (base sizing), buyback level, conviction score (excl. P/E to avoid double-counting), and sector caps (40% max per sector). Assets with conviction <20 receive zero weight.
This is an illustrative model portfolio based on live fundamentals and ARCA-style rules. It is NOT investment advice.

Dynamic Strike Price — P/E = 10×
Computed in real-time from live DefiLlama + CoinGecko data
The Strike Price is the P/E = 10× reference price. Signals are based directly on the live P/E zone; the Strike line visualizes that boundary. It is dynamically computed: Strike = (Annualized Revenue ÷ Circulating Supply) × 10. When price is below strike, the asset trades at P/E < 10× — aggressive buy zone. When above, P/E increases proportionally through Fair Value → Growth → Premium → Overheated zones. All P/E calculations use Circulating Market Cap (price × circulating supply), not FDV.
🔴 LIVE = Computed from DefiLlama revenue + CoinGecko circulating supply · 📋 STATIC = Manual estimate when live data is unavailable
P/E = Circ. MC ÷ Protocol Revenue (30d annualized) · P/S = Circ. MC ÷ Total Fees (30d annualized). Revenue = what the protocol captures. Fees = total paid by users (includes LP share).
